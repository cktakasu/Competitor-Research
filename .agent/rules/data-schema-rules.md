# データ構造に関するルール

## 鉄則
- 遮断容量は「IEC 60947-2 (Icu, kA単位)」と「IEC 60898-1 (Icn, A単位)」を
  **必ず別フィールド**として保持する。混在させない
- 電圧レベル別（220-240V, 380-415V, 440V）に遮断容量の値を分けて格納する
- 数値データは文字列にせず、必ず number 型で格納する
  （例: `"icu_380_415v_ka": 20` であり `"icu_380_415v_ka": "20kA"` ではない）
- `null` は「そのスペックが存在しない・該当しない」を意味する
- データが未調査の場合は `null` ではなく当該フィールド自体を省略する

## メーカー拡張時のルール
- 新メーカー追加時は既存のJSONスキーマに完全準拠する
- メーカー固有のフィールド追加が必要な場合は `manufacturer_specific` オブジェクト内に格納する
- 共通フィールドの意味を変えてはならない
